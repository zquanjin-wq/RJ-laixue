### pnpm build ⚠️ 环境阻塞
```
Error: Initiated Worker with invalid NODE_OPTIONS env variable:
--use-system-ca is not allowed in NODE_OPTIONS
    at ignore-listed frames { code: 'ERR_WORKER_INVALID_EXEC_ARGV' }
```

**根因**（P0-1b 定位确认）：环境变量 `NODE_OPTIONS` 由 **WorkBuddy 桌面端注入**：

```
$NODE_OPTIONS = --require="C:/Users/ruijie/AppData/Local/Programs/WorkBuddy/resources/app.asar.unpacked/cli/vendor/shim/genie-safe-delete.cjs" --use-system-ca
```

**与项目代码无关**——不是 shell profile / .npmrc / package.json scripts / CI 配置。

**解法（P0-1b 实测）**：

```bash
# 方式 1：临时 unset（推荐开发用）
NODE_OPTIONS="" npx next build
# → ✓ Compiled successfully in 22.2s  ✅

# 方式 2：保留 WorkBuddy 自有 --require，证书走专用变量
NODE_OPTIONS=--require=".../genie-safe-delete.cjs" \
  NODE_EXTRA_CA_CERTS=<企业 CA 路径> \
  pnpm build

# 方式 3：Vercel Dashboard 永久配置
# Project Settings → Environment Variables → 把 NODE_OPTIONS 设成
#   --require=".../genie-safe-delete.cjs"（仅保留需注入的 shim，扔掉 --use-system-ca）
```

**需要解决**：
- 在 Vercel Dashboard → Project Settings → Environment Variables 找到 `NODE_OPTIONS`，从值里删掉 `--use-system-ca`（或整个变量）
- 或在本地 build 前 `unset NODE_OPTIONS` / `set NODE_OPTIONS=` （Windows）

Vercel preview 冒烟因此无法在本地进行，**需用户在 Vercel 平台确认或修复环境变量后，冒烟测试通过验证步骤 6**。

## 五、流程纪律新规约（已写入 PROJECT-STATE.md）

> 今日第二次"未固化的工作差点/真的丢失"——上次是工作树里的开关改动被 reset 清掉（独立评审发现并修正），这次是分支上的 commit 67e92657 被 reset 清掉（用户发现后重做）。

**铁律**：

> `git reset --hard` 前必须自查：
> 1. 当前分支是否有未合入 main 的 commit？→ 先 merge 或打 tag
> 2. 工作树是否有未提交改动？→ 先 `git stash` 或 commit
> 3. 两者任一存在，**不** reset
> 
> 例外：明确知道要丢弃的废弃分支，先 `git branch -D <branch>` 删分支，不需要 reset。

## 六、遇到的问题

1. **初次实现位置错误**：LEGACY_AUTOSAVE 开关最初加在 `fireAndForgetAutoSave` 函数**内部**——独立评审发现 `saveStageToCloud` 有 3 个调用点（自动/顺序修复/手动按钮），开关应该只在调用点管"自动"那条。**修正后**：开关上移到 hook 内的 2 个 `fireAndForgetAutoSave(stage.id)` 调用点，`fireAndForgetAutoSave` 函数回归纯净。已 commit `21712c39`。

2. **cp/ssrf-1 分支被 reset 清掉**：第一次 cherry-pick 7c2aaafc 提交到 cp/ssrf-1 分支（commit `67e92657`），随后 `git reset --hard main` 把该 commit 变成悬空对象。**修正**：重做 cherry-pick 这次直接用真 commit（不是 `--no-commit`）落到 main（`6d7142a1`）。

3. **pnpm build 阻塞**（见 §三）——环境配置问题，不属于本次任务。
