## 二、关于几个最受关注的问题（集中回应）

收到反馈后，我们梳理出几个"被反复问到"的问题，统一回应如下：

### 问题 1：大模型是公有部署还是公司内部私有部署？（来自 Cindy 等同事）

**现状**：当前底层调用的是合规商用 API，**数据通过 HTTPS 加密传输，且不参与任何模型训练**——你们的内容、案例、术语不会被用于模型迭代。

**下一步**：**我们已经在推进私有化部署**，预计 Q3 上线。届时模型推理完全在公司内网完成，数据不出锐捷。

### 问题 2：AI 能否准确理解专业术语和产品逻辑？还是需要大量人工精修？（来自 Grant 等同事）

**这是关键问题**，我们直接面对：

- 通用术语准确度约 **85%**，日常沟通够用；
- **锐捷专业术语（产品型号、参数、命令）目前约 60%**，确实需要补充。

**解决方案**分两步：
- **Q3 上线**：每门课件允许附加"术语对照表"，AI 优先用表内术语；
- **Q4 上线**：把锐捷产品手册、解决方案文档、���业案例接入知识库（RAG），准确度提升到 **95% 以上**。

### 问题 3：英文课件生成失败（来自吕胜蛟 等同事）

**这是一个真实 Bug**——我们的多语言 pipeline 当前只支持"英文材料识别 + 中文课件生成"，但**反向流程（中文→英文）有缺陷**。

**修复时间：Q3 内**，修复后会在群里通知。**给遇到问题的同事带来的不便，我们表示抱歉。**

## 三、关于后续安排（提前预告）

大家问得最多的不是"这个工具好不好"，而是"你们下一步怎么走"。我们把接下来要做的事按优先级提前预告，方便大家提前规划：

### 🔴 第一阶段（1-3 个月，重点是合规与基础）

| 事项 | 解决什么 |
|---|---|
| **私有化部署** | 数据不出公司，合规过审 |
| **飞书 SSO + 组织架构同步** | 统一账号体系，接入门槛降到最低 |
| **大模型私有化推理服务** | 配套私有化部署 |
| **英文课件生成 Bug 修复** | 回应吕胜蛟老师 |
| **内容审核 + 敏感词过滤** | 合规硬要求 |

### 🟡 第二阶段（3-6 个月，重点是产品力）

| 事项 | 价值 |
|---|---|
| **锐捷专业术语库 + RAG** | 解决 AI 术语准确度 |
| **岗位模板**（销售/研发/客服/产研/审计） | 一键生成岗位专属课件 |
| **课件→PPT/讲稿导出** | 视频能力的短期方案 |
| **第三方平台集成 API**（北森/eLearning） | 和现有培训系统打通 |
| **考试功能** | 学习闭环 |

### 🟢 第三阶段（6-12 个月，重点是差异化）

- AI 数字人 + 文本→视频（视频化能力）
- 行业垂直版本（运营商 / 金融 / 教育）
- 学习路径 + 进度跟踪（体系化）
- **经验库 / 案例库**（沉淀一线实战经验）
- **国际化（5+ 语言）**——回应战略合作部 James 老师
- 课件市场（UGC 飞轮）
- 社会化学习（学员讲师互动）
